Rendering Workflow of Slate

Photo by Memories on 35mm The rendering of Slate happens in FSlateApplication::PrivateDrawWindows. It first iterates every visible window to collect nessary rendering information used for rendering. Then consumes these information by invoking corresponding platform’s graphics API (D3D, OpenGL, etc.) to do the rendering job. Collecting A window (SWindow) in UE4’s terminology is the top-level widget of game, usually one game has only one window, every widget we created at runtime is just a child of it.
